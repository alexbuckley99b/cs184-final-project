Font Rasterization

Kyle Wallace, Raiya Choudhuri, Scott Durand, Alex Buckley


Summary

In our project we intend to create an efficient system for rasterizing the font characters in a TrueTypeFont (.ttf) file. Next we hope to implement a simple texture-sourced flood fill into the rasterized area. Afterwards, we hope to create a physics simulation to model collisions between our character objects.



Problem Description

So far in this class, we have not discussed a method for rasterizing non-triangular shapes. We hope to overcome this by designing an algorithm that rasterizes generic closed composite bezier curves, namely, fonts. We also hope to implement texture mapping to render the flood fill of a font character from a texture source. Afterwards, we hope to challenge ourselves by implementing a physics simulation, treating each character as a separate physical object.

Fonts that are parameterized by curves are more accurate and scalable than bitmap or triangle-based fonts are. Determining how to rasterize and fill these fonts is important for modern rendering of text.

Right now we intend to implement font rasterization by performing intersection tests between horizontal lines and the bezier curves that comprise the font character. Using this information, we can traverse through the pixels, marking them as inside or outside of the character based on how many bezier curve intersections there are to the pixel’s left (an odd number of intersections means that this pixel is inside of the character, and an even number means that it is outside). To prevent jaggies / aliasing, we will utilize supersampling, as we did with triangles. When considering the efficiency of our code, we may limit supersampling to only those pixels near the edges of our rasterized character. We also intend to explore thread and GPU-based methods for parallelizing our code.



Goals and Deliverables

Given a .ttf file, an input string (and potentially other options such as positioning, alignment, size, etc.), we will create images that display characters in TrueType font, which are made of quadratic Bézier curves.

Define how you will measure the quality / performance of your system

We will measure the quality of our output via its likeness to existing images produced using the font at different resolutions / scales. We will measure performance via timed benchmarking

What we plan to deliver

- Given a font as a .ttf file, rasterize an input string.

- Render character outlines given TTF fonts.

- Render flood filled characters given TTF fonts.

- Make this process efficient by exploiting parallelism (GPU / TLP)

What we plan to deliver

- Fill characters using pixel samples from a source texture file.

- Create a physics simulation for closed bezier outlines.
